01

Das richtige Modell wählen

Opus für 90% aller Aufgaben zu nutzen ist wie mit einem Ferrari einkaufen zu fahren. Sonnet 4.6 ist nicht das „günstige" Modell — es ist das optimale für fast alles.

EinsatzVor jeder neuen Aufgabe — 10 Sekunden Entscheidung
ErsparnisBis zu 50% weniger Token-Verbrauch
MODELL CHEAT-SHEET Haiku → Kurze Zusammenfassungen → Einfache Kategorisierungen → Schnelle Ja/Nein-Fragen → Text kürzen oder formatieren Sonnet 4.6 ← Dein Standard (90% aller Aufgaben) → Texte schreiben & überarbeiten → Code generieren und debuggen → Recherche und Analyse → Komplexe Erklärungen → Brainstorming Opus 4 (nur wirklich nötig für) → Mehrstufige Planungsaufgaben → Widersprüchliche Anforderungen auflösen → Hochkomplexe Logik / Mathematik
02

Klarer Auftrag — keine Korrekturschleifen

Jede Korrekturschleife kostet 3× so viel wie ein präziser erster Auftrag. Die meisten Token-Verluste passieren beim Nachbessern. Dieses Template verhindert das.

EinsatzAls Standard-Template für jeden Auftrag
ErsparnisBis zu 40% weniger durch eliminierte Loops
AUFTRAGS-TEMPLATE (Token-effizient) Aufgabe: [ein klares Verb + Objekt] Format: [Stichpunkte / Code / Fließtext / Tabelle] Länge: [kurz / max. 200 Wörter / eine Seite] Kontext: [nur was direkt relevant ist] Ausgabe: [was genau, was NICHT drin sein soll] — Schlecht: „Mach mir irgendwas über KI" Gut: „Schreibe 3 Stichpunkte (max. 15 Wörter je) warum Claude besser als ChatGPT für Code ist."
03

Projekt-Dateien als Kontext-Speicher

Ohne Projekt-Dateien erklärst du Claude bei jeder neuen Konversation wer du bist, was du baust und wie du arbeitest. Eine einzige Datei stoppt das — dauerhaft.

EinsatzEinmal einrichten, bei jeder Session automatisch aktiv
Ersparnis50–200 Token pro Session — täglich
PROJEKT-DATEI TEMPLATE Dateiname: kontext.md # Wer ich bin [1 Satz: Name, Beruf, Fokus] # Was ich gerade baue [2-3 Sätze: Projekt, Ziel, Status] # Mein bevorzugter Stil - Antworten: kurz und direkt, kein Smalltalk - Format: [Stichpunkte / Fließtext / Code] - Sprache: Deutsch # Wiederkehrende Infos - Website: [URL] - Zielgruppe: [wer] - Wichtige Begriffe: [...] → Claude liest die Datei automatisch bei jedem Start

01

Das richtige Modell wählen

Sonnet 4.6 für 90% — Opus nur für echte Komplexität.

Haiku → Zusammenfassen, Formatieren, Kategorisieren Sonnet 4.6 → Alles andere (Standard) Opus → Nur: komplexe Planung, Widersprüche lösen
02

Input vorfiltern

Ungefilterte Gedanken verbrennen Token bevor Claude antwortet. Wispr Flow oder ChatGPT als Vorfilter nutzen.

Workflow: 1. Gedanken in Wispr Flow einsprechen 2. ChatGPT o4-mini als Vorfilter: „Forme diesen Text in einen präzisen Auftrag für Claude um. Max. 100 Wörter. [dein Text]" 3. Erst dann zu Claude senden
03

Skill statt Trick

Ein einmal gebauter Skill spart dauerhaft mehr als jeder Caveman-Mode-Hack.

In Claude Code einen Skill anlegen: /skill create [name] Oder: Projekt-Anweisung mit Standard-Workflow → Claude führt ihn automatisch aus → Keine repetitiven Erklärungen mehr nötig
04

Neue Konversation starten

Je länger eine Konversation, desto teurer jede Antwort — Context trägt sich auf.

Starte neu wenn: → Mehr als 5-6 Nachrichten hin und her → Claude anfängt Dinge zu „vergessen" → Das Thema wechselt → Antworten werden länger und umständlicher Vorher: „Fasse den Stand in 3 Sätzen zusammen" → Neue Session mit dieser Zusammenfassung starten
05

Klarer Auftrag = keine Loops

Fünf Felder die Korrekturschleifen eliminieren.

Aufgabe: [Verb + Objekt] Format: [Stichpunkte / Code / Fließtext] Länge: [max. X Wörter] Kontext: [nur das Relevante] Ausgabe: [was genau, was NICHT]
06

Projekt-Dateien nutzen

Einmal schreiben, bei jeder Session automatisch aktiv. Kein Re-Erklären mehr.

kontext.md anlegen mit: → Wer ich bin (1 Satz) → Was ich gerade baue (2-3 Sätze) → Bevorzugter Stil + Format → Wiederkehrende Infos Claude liest sie automatisch → kein Wiederholen nötig

Bonus

4 Bonus-Hebel

Für alle die noch tiefer optimieren wollen.

01

Kürze als Projekt-Anweisung

Claude neigt dazu ausführlich zu sein. Eine einzige Zeile in der Projekt-Anweisung ändert das dauerhaft.

In deine Projekt-Anweisung einfügen: „Antworte immer so kurz wie möglich. Kein Smalltalk. Keine Wiederholungen. Direkt zur Lösung. Wenn ich mehr will, frage ich."
02

Extended Thinking dosiert

Claudes „Nachdenken" kostet extra Token — für einfache Aufgaben ist es reine Verschwendung.

Extended Thinking NUR einschalten für: → Komplexe Mathematik / Logik → Mehrstufige Planung mit Widersprüchen → Strategische Entscheidungen Für alles andere: deaktiviert lassen. Spart 30–50% der internen Verarbeitungs-Token.
03

Summarize & Continue

Wenn eine Konversation zu lang wird, rettest du den Kontext in 10 Sekunden.

1. Am Ende einer langen Session: „Fasse den bisherigen Stand in 3 Sätzen zusammen. Was haben wir beschlossen? Was ist offen?" 2. Neue Konversation starten 3. Erste Nachricht: „Kontext: [deine 3-Satz-Zusammenfassung] Weiter mit: [nächster Schritt]"
04

Micro-Tasks statt Mammut-Aufträge

Große Aufgaben in kleine Chunks aufteilen — kleinere Outputs, weniger Token, weniger Fehler.

Statt: „Bau mir eine komplette Landing Page" Besser aufteilen: 1. „Erstelle nur die Struktur (7 Sections, 1 Satz je)" 2. „Schreibe nur den Hero-Text" 3. „Jetzt die Features-Section" Kleiner Auftrag → kürzere Antwort → weniger Token
